# 湃乐多邮件营销系统 — 配置与启动说明

本文档说明**开发环境**、**云端 Linux 公网 IP 访问**及**生产环境**下的配置方式及启动指令。

---

## 一、开发环境（本机）

### 1. 后端配置（`backend/.env`）

在 `backend` 目录下复制 `.env.example` 为 `.env`，按需修改：

```bash
cd backend
# 若没有 .env，可复制示例（示例中为注释，需去掉 # 并填值）
copy .env.example .env   # Windows
# cp .env.example .env   # Linux / macOS
```

**开发时常用项：**

| 变量 | 说明 | 开发建议 |
|------|------|----------|
| `DATABASE_URL` | 数据库连接 | 默认 `sqlite:///./email_marketing.db` 即可 |
| `JWT_SECRET` | JWT 密钥 | 可保持默认，生产必须修改 |
| `DEBUG` | 调试模式 | 可选 `True`，便于排查问题 |
| `CORS_ORIGINS` | 允许的前端来源 | 默认 `http://localhost:5173,http://127.0.0.1:5173` |
| `AI_API_BASE_URL` / `AI_API_KEY` | AI 接口 | 不填则邮件预览为占位文案；要真实生成需填写 |
| `SMTP_*` | 发邮件 | 开发可不配，发邮件会报错 |
| `LOG_DIR` | 应用日志目录 | 默认 `logs`，日志按日期写入 `backend/logs/YYYY-MM-DD.log` |

### 2. 后端启动（开发）

```bash
cd backend
# 建议使用虚拟环境
python -m venv .venv
.venv\Scripts\activate   # Windows
# source .venv/bin/activate  # Linux / macOS

pip install -r requirements.txt   # 若无则安装 fastapi uvicorn 等
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

- 后端地址：`http://127.0.0.1:8000`
- `--reload`：代码改动自动重启（仅开发使用）

### 3. 前端配置（开发）

开发时前端通过 Vite 代理访问后端，**一般无需额外配置**：

- 请求 `/api`、`/uploads` 会被代理到 `http://127.0.0.1:8000`
- API 基地址默认 `/api`（见 `frontend/src/api/client.js`）

若后端不在本机 8000 端口，可增加环境变量（项目根下建 `frontend/.env.development`）：

```env
VITE_API_BASE_URL=http://你的后端地址:8000/api
```

### 4. 前端启动（开发）

```bash
cd frontend
npm install
npm run dev
```

- 前端地址：`http://localhost:5173`
- 浏览器访问该地址即可，接口会走代理到后端

### 5. 开发环境启动顺序小结

1. 先启动**后端**：`cd backend` → `uvicorn app.main:app --reload --host 127.0.0.1 --port 8000`
2. 再启动**前端**：`cd frontend` → `npm run dev`
3. 浏览器打开：`http://localhost:5173`

## 二、云端 Linux 服务器 + 公网 IP 访问（另一台机器浏览器访问）

代码部署在一台**云端 Linux 服务器**上，在**另一台电脑**通过公网 IP（例如 `101.132.22.43`）用浏览器访问。

### 1. 服务器上需要做的配置

#### 1.1 后端 `backend/.env`

- 后端会对外提供 API，需允许「前端页面所在来源」跨域，并建议改掉默认 JWT 密钥：

```env
# 允许来自公网 IP 的前端页面访问（端口与下面前端实际使用一致）
CORS_ORIGINS=http://101.132.22.43:5173,http://127.0.0.1:5173

# 生产建议修改
JWT_SECRET=请改为随机强密钥
DEBUG=False
```

若前端用 Nginx 挂在 80 端口，则增加：`http://101.132.22.43`（以及 `http://101.132.22.43:80` 如需要）。

#### 1.2 前端构建时的 API 地址（重要）

前端页面是在**另一台机器的浏览器**里打开的，请求会从那边发到**服务器的公网 IP**，所以构建时要填**后端公网地址**：

在 `frontend` 目录下建 `.env.production`：

```env
# 填服务器公网 IP + 后端端口（与下面启动后端用的端口一致）
VITE_API_BASE_URL=http://101.132.22.43:8000/api
```

若你有域名并做了解析，可写成：`VITE_API_BASE_URL=https://你的域名:8000/api`。

### 2. 服务器防火墙 / 安全组

在云控制台和本机防火墙中放行：

- **8000**：后端 API
- **5173** 

### 3. 在服务器上启动服务

#### 方式 A：前端用开发服务器（适合调试、临时用）

```bash
# 终端 1：后端（在项目 backend 目录下）
cd /path/to/email-marketing/backend
source .venv/bin/activate   # 或 python3 -m venv .venv 后 activate
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

```bash
# 终端 2：前端 dev，并允许外网访问（在项目 frontend 目录下）
cd /path/to/email-marketing/frontend
npm run dev -- --host 0.0.0.0
```

- 前端默认端口 5173，需在 `CORS_ORIGINS` 里已包含 `http://101.132.22.43:5173`。
- 另一台机器浏览器访问：**http://101.132.22.43:5173**

#### 方式 B：前端构建后本地预览服务（推荐，更接近生产）

```bash
# 1）构建前端（只需执行一次，或代码/配置改后再执行）
cd /path/to/email-marketing/frontend
# 确保已有 .env.production 且 VITE_API_BASE_URL=http://101.132.22.43:8000/api
npm run build

# 2）后端（可先在后端目录起好）
cd /path/to/email-marketing/backend
uvicorn app.main:app --host 0.0.0.0 --port 8000

# 3）在 frontend 目录另开终端：用 Vite 预览已构建产物，并对外网开放
cd /path/to/email-marketing/frontend
npx vite preview --host 0.0.0.0 --port 4173
```

- 另一台机器浏览器访问：**http://101.132.22.43:4173**
- 此时 CORS 需包含 `http://101.132.22.43:4173`（上面已在示例中写出）。

### 4. 在另一台机器上访问

- 浏览器地址栏输入：
  - 方式 A：`http://101.132.22.43:5173`
  - 方式 B：`http://101.132.22.43:4173`
- 若无法打开：先确认服务器上后端、前端进程在跑，再检查云安全组和本机防火墙是否放行对应端口。

### 5. 小结（公网 IP 101.132.22.43 示例）

| 步骤 | 说明 |
|------|------|
| 后端 .env | `CORS_ORIGINS` 包含 `http://101.132.22.43:5173` 
| 前端 .env.production | `VITE_API_BASE_URL=http://101.132.22.43:8000/api` |
| 防火墙/安全组 | 放行 8000、5173 或 4173 |
| 后端启动 | `uvicorn app.main:app --host 0.0.0.0 --port 8000` |
| 前端 | 方式 A：`npm run dev -- --host 0.0.0.0` → 访问 **:5173**；方式 B：`npm run build` + `npx vite preview --host 0.0.0.0 --port 4173` → 访问 **:4173** |

## 三、生产环境（域名 + Nginx 等）

### 1. 后端配置（`backend/.env`）

生产环境**必须**单独维护一份 `.env`（不要与开发混用），并至少修改以下项：

| 变量 | 说明 | 生产要求 |
|------|------|----------|
| `JWT_SECRET` | JWT 签名密钥 | **必须**改为随机强密钥 |
| `DEBUG` | 调试模式 | **必须**为 `False` |
| `DATABASE_URL` | 数据库 | 若用 MySQL/PostgreSQL，改为对应连接串 |
| `CORS_ORIGINS` | 允许的前端来源 | 填生产前端域名，如 `https://your-domain.com` |
| `SMTP_HOST` / `SMTP_USER` / `SMTP_PASSWORD` / `SMTP_SENDER` | 发件 | 填真实 SMTP，否则无法发邮件 |
| `AI_API_BASE_URL` / `AI_API_KEY` | AI 接口 | 按需填写，否则邮件预览为占位 |
| `UPLOAD_DIR` | 上传目录 | 可按部署路径改为绝对路径 |

### 2. 后端启动（生产）

**不**使用 `--reload`，并建议绑定到 `0.0.0.0` 以便对外访问：

```bash
cd backend
# 使用生产用虚拟环境
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

- 多进程示例（按需使用，注意与定时任务兼容）：
  ```bash
  uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 2
  ```
- 若通过 Nginx 等反向代理，可只监听本机：`--host 127.0.0.1 --port 8000`，由 Nginx 转发到 80/443。

### 3. 前端配置（生产）

生产环境前端通常**单独构建**并部署到静态服务器或 CDN，需要让请求发到**真实后端地址**：

在 `frontend` 目录下建 `.env.production`（或构建时传入环境变量）：

```env
VITE_API_BASE_URL=https://你的后端域名/api
```

例如后端为 `https://api.example.com`，则填：

```env
VITE_API_BASE_URL=https://api.example.com/api
```

### 4. 前端构建与预览（生产）

```bash
cd frontend
npm install
npm run build
```

- 产物在 `frontend/dist`，将该目录部署到 Nginx、Caddy 或任意静态托管即可。
- 本地验证构建结果：
  ```bash
  npm run preview
  ```
  会启动静态服务（默认如 `http://localhost:4173`），此时需保证 `VITE_API_BASE_URL` 指向可访问的后端。

### 5. 生产环境部署顺序小结

1. **后端**：在服务器上配置好 `backend/.env`，执行  
   `uvicorn app.main:app --host 0.0.0.0 --port 8000`
2. **前端**：配置 `frontend/.env.production` 中的 `VITE_API_BASE_URL`，执行  
   `npm run build`，将 `frontend/dist` 部署到 Web 服务器
3. **反向代理**（推荐）：用 Nginx 等将前端域名指向 `dist`，将 `/api`、`/uploads` 反向代理到后端 `http://127.0.0.1:8000`，这样前端可继续用相对路径 `/api`，无需改 `VITE_API_BASE_URL`

## 四、环境变量速查

### 后端（`backend/.env`）

| 变量 | 默认/说明 |
|------|-----------|
| `DATABASE_URL` | `sqlite:///./email_marketing.db` |
| `JWT_SECRET` | 生产必改 |
| `DEBUG` | `False` |
| `SMTP_HOST` / `SMTP_PORT`(465) / `SMTP_USER` / `SMTP_PASSWORD` / `SMTP_SENDER` | 发邮件用 |
| `AI_API_BASE_URL` / `AI_API_KEY` / `AI_API_MODEL`(gpt-4o-mini) | AI 文案 |
| `CORS_ORIGINS` | 逗号分隔的前端来源 |
| `UPLOAD_DIR` | `uploads` |

### 前端（构建时）

| 变量 | 说明 |
|------|------|
| `VITE_API_BASE_URL` | 不填时开发用 `/api`（走 Vite 代理）；生产填完整后端 API 根地址，如 `https://api.example.com/api` |

## 五、常用启动命令汇总

| 场景 | 后端 | 前端 | 访问方式 |
|------|------|------|----------|
| **本机开发** | `uvicorn app.main:app --reload --host 127.0.0.1 --port 8000` | `npm run dev` | 本机 http://localhost:5173 |
| **云端 Linux，公网 IP 访问** | `uvicorn app.main:app --host 0.0.0.0 --port 8000` | `npm run dev -- --host 0.0.0.0` 或 `npm run build` + `npx vite preview --host 0.0.0.0 --port 4173` | 他机 http://公网IP:5173 或 :4173 |
| **生产（域名+Nginx）** | `uvicorn app.main:app --host 0.0.0.0 --port 8000` | `npm run build`，部署 `dist` | 域名 |
